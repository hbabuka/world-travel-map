World Travel Map is a React + Vite app for tracking countries visited, logging trips, and visualizing them on an interactive D3 world map.

**Stack:** React + Vite (v8), Tailwind CSS v4 (@tailwindcss/vite plugin), shadcn/ui primitives (Radix UI), D3 + topojson-client + world-atlas, Supabase (auth + DB).

**Auth:** Supabase Google OAuth. `useAuth` hook in `src/hooks/useAuth.js`. Env vars: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` (see `.env.example`).

**DB table:** `trips` — fields: id, user_id, title, country_code (ISO alpha-2), country_name, start_date, end_date, notes, photo_urls, created_at.

**Key conventions:**
- Path alias `@/` → `src/`
- CSS design tokens live in `src/index.css` under `@theme {}` (Tailwind v4 syntax)
- Raw CSS custom properties (e.g. `--ocean`, `--visited`) are re-exported from `:root {}` for D3 consumption
- UI primitives in `src/components/ui/` are hand-rolled shadcn-style (not generated by the shadcn CLI)
- Map state (selected country, zoom, modal open) lives in `MapStoreContext` (`src/store/mapStore.jsx`)
- `useTrips` hook manages all CRUD + derives `visitedCountryCodes` set

**Why:** Design files are in a Claude Project at claude.ai and shared iteratively during the build.

**How to apply:** When adding new components, follow the existing pattern: Radix primitives + `cn()` for class merging, no external CSS files, design tokens via CSS custom properties.
